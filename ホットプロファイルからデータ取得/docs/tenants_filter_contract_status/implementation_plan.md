# テナントマスタの契約ステータスによるフィルタリング実装計画

## 目標
テナントマスタシートの「契約ステータス」列を確認し、「解約済み」となっているレコードを企業情報の取得対象から除外する。「本契約」および「空欄」は対象とする。

## 変更内容

### 1. `企業情報.js` の `getTenantMasterData` 関数

#### A. ヘッダエイリアスの追加
- `headerAliases` オブジェクトに `contractStatus` を追加し、見出し名の候補（"契約ステータス"）を定義する。

#### B. 列番号の解決
- `resolveColumnIndex` を使用して、「契約ステータス」列のインデックスを取得する `colContractStatus` を定義する。
- 必須項目ではないため、列が見つからない場合のエラー送出は行わない（見つからない場合は全件対象）。

#### C. データ読み取り範囲の追加
- `colContractStatus` が有効な場合、その列のデータを取得する `contractStatuses` 配列を用意する。

#### D. ループ処理内のフィルタリングロジック追加
- 各行の処理において、契約ステータスの値をチェックする。
- 値が「解約済み」（前後の空白除去後）の場合、その行をスキップする (`continue`)。
- 「本契約」や空文字、その他の値の場合は処理を継続する。

## 検証計画
1. テナントマスタに以下のパターンのレコードを用意する。
   - ステータス: "本契約" (対象)
   - ステータス: "解約済み" (除外対象)
   - ステータス: "" (空欄) (対象)
   - ステータス: "トライアル" (対象 - 要件になければ除外しない)
2. スクリプトを実行し、企業情報シートに出力されるレコードを確認する。
3. "解約済み" のレコードのみが出力されていないことを確認する。
