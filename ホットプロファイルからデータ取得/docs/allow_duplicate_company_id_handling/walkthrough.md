# ウォークスルー: テナントマスタ重複会社ID対応

## 概要
テナントマスタシートで同じ「ホットプロファイル会社ID」が複数行存在しても、「テナント_顧客ID」が異なれば企業情報シートに別々のレコードとして登録・更新されるよう改修しました。

## 変更ファイル

### 1. `企業情報.js`

#### A. `generateCompositeKey` 関数の追加 (行 16-29)
会社IDとテナント顧客IDを組み合わせた一意なキー文字列を生成するヘルパー関数を追加しました。
- 形式: `"会社ID_テナント顧客ID"` (例: `"12345_ABC"`)
- テナント顧客IDの先頭 `'` は正規化時に除去

#### B. `buildExistingDataMapByHeader` の修正 (行 491-538)
- 「テナント_顧客ID」列も読み込むよう変更
- マップのキーを会社ID単体から複合キーに変更
- これにより、同じ会社IDでもテナント顧客IDが異なる行は別エントリとして管理

#### C. `upsertClients` の修正 (行 562-783)
- `tenants.find()` を `tenants.filter()` に変更し、同一会社IDに紐づく**すべてのテナントレコード**を取得
- 各テナントレコードに対してループ処理を行い、それぞれ別の行として登録/更新
- `processedIds` のチェックを複合キーベースに変更（二重処理防止）
- `existingDataMap` の検索・更新も複合キーを使用

---

### 2. `ラストタッチ確認.js`

#### A. `rowMap` の値を配列に変更 (行 562-592)
- 従来: `Map<会社ID, {rowIndex, name}>`
- 変更後: `Map<会社ID, [{rowIndex, name}, ...]>`
- 同じ会社IDの複数行を保持可能に

#### B. `updatesMap.forEach` 内のループ処理追加 (行 626-671)
- `entries.forEach()` を追加し、該当するすべての行にラストタッチを更新

## 動作イメージ

```
【テナントマスタ】
| ホットプロファイル会社ID | テナント_顧客ID |
|--------------------------|-----------------|
| 12345                    | 'A001           |
| 12345                    | 'B002           |

【企業情報シート（更新後）】
| 会社ID | 新会社ID | テナント_顧客ID | ... |
|--------|----------|-----------------|-----|
| 12345  | 12345    | 'A001           | ... |
| 12345  | 12345    | 'B002           | ... |
```

## 検証方法
1. テナントマスタに重複する会社IDを持つ異なるテナント顧客IDのレコードを用意
2. 週次または日次更新を実行
3. 企業情報シートに該当する会社IDのレコードがテナント顧客ID別に複数行作成されることを確認
4. 再度更新を実行し、二重登録されず正しく更新されることを確認
