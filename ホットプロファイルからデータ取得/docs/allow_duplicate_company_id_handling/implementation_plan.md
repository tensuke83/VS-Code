# テナントマスタの重複会社ID対応計画

## 目標
テナントマスタシートで「ホットプロファイル会社ID」が重複していても、「テナント_顧客ID」が異なれば、企業情報シートに別々のレコードとして登録・更新できるようにする。

## 背景
現在は `upsertClients` 関数内で `tenants.find(...)` を使用してテナント情報を検索しているため、同じ会社IDを持つレコードが複数あっても最初の1件しか処理されない。また、重複チェック（`existingDataMap`）も会社ID単体で行われているため、同一会社IDの別レコードを区別できない。

## 変更内容

### 1. `企業情報.js` の改修

#### A. 複合キー生成ヘルパーの追加
会社IDとテナント顧客IDを組み合わせた一意なキーを生成する関数 `generateCompositeKey` を追加する。
- テナント顧客IDに含まれる可能性のある先頭のシングルクォート `'` は除去して正規化する。

#### B. `buildExistingDataMapByHeader` の修正
- 既存データのマップ作成時に、「テナント_顧客ID」列も読み込む。
- マップのキーを会社ID単体から、`generateCompositeKey` で生成した複合キーに変更する。
  - キー1: `generateCompositeKey(会社ID, テナント顧客ID)`
  - キー2: `generateCompositeKey(新会社ID, テナント顧客ID)` (新会社IDがある場合)

#### C. `upsertClients` の修正
- `tenants.find` を `tenants.filter` に変更し、特定の会社IDに紐づく**すべての**テナント情報の配列を取得する。
- 取得したテナント情報配列に対してループ処理を行い、それぞれについて以下を実施する。
  - 行データの作成（テナント顧客IDは各テナント情報のものを使用）。
  - `existingDataMap` からの検索にも複合キーを使用する。
  - 新規追加か更新かを判定し、リストに追加する。

### 2. `ラストタッチ確認.js` の改修 (推奨)
`updateLastTouchDailyDiffStep`（日次差分更新）において、`rowMap` が会社IDをキーにしているため、重複IDの一方の行しか保持できない問題がある。
- `rowMap` の値を `Object` から `Array<Object>` に変更し、同一IDの複数行を保持できるようにする。
- 更新ロジックで、該当するすべての行に対して更新を適用するようにループ処理を追加する。

## 検証計画
1. テナントマスタに重複する会社ID（例: ID 100）を持つ異なるテナント顧客ID（例: "A", "B"）のレコードを用意する。
2. スクリプトを実行し、企業情報シートに ID 100 のレコードが2行作成され、それぞれ "A", "B" の顧客IDを持っていることを確認する。
3. 再度実行し、データが正しく更新（二重登録されず、更新日時などが更新される）されることを確認する。
