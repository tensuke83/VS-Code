# Teams通知レイアウト改善 実装計画

Teamsへの通知をAdaptive Cardの機能（FactSet, ColumnSetなど）を活用して、より見やすく構造化されたレイアウトに変更します。

## ユーザーレビューが必要な事項
- 通知の見た目が大きく変わるため、変更後のレイアウトが意図通りか確認が必要です。

## 変更内容

### `共通.js`

#### 機能拡張: `sendTeamsAdaptiveCard`
- `payload` に `body` プロパティが含まれている場合、それをそのまま Adaptive Card の `body` として使用するように変更します（既存の `title` + `markdown` 形式も互換性のため維持）。

#### 機能刷新: `safeNotifySummary`
- 従来のMarkdown文字列生成をやめ、Adaptive Cardのオブジェクト構造を直接生成します。
- **全体サマリ:**
  - 処理開始時刻、終了時刻、総所要時間を `FactSet` で表示し、ラベルと値をきれいに整列させます。
- **対象別サマリ:**
  - 各対象（企業情報、ラストタッチ等）ごとに `Container` を作成。
  - 数値情報（追加、更新、見つからない、合計）を `FactSet` で表示。
  - 重要度に応じて色分け（成功はDefault/Good、エラー等はAttention）を行います。

## 検証計画

### 動作確認
1.  スクリプトを手動実行（またはテスト用関数を作成して実行）し、Teamsに通知を送信します。
2.  Teams上で通知カードが表示されることを確認します。
    - 全幅表示になっているか（前回対応済み）。
    - 各項目が整列しているか。
    - エラー時の赤字表示などが機能しているか。
